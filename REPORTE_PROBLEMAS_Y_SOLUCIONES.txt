     class PaginatedResponse(BaseModel, Generic[T]):

        page: int
        limit: int
        pages: int
        has_next: bool
        has_prev: bool
  
  2. Crear base repository con paginación:
     
     def paginate(self, page: int = 1, limit: int = 20):
        offset = (page - 1) * limit
        total = self.session.query(self.model).count()
        items = self.session.query(self.model)\
            .offset(offset)\
            .limit(limit)\
            .all()
        
        return {
            "data": items,
            "total": total,
            "page": page,
            "limit": limit,
            "pages": (total + limit - 1) // limit,
            "has_next": page < pages,
            "has_prev": page > 1
        }

PROBLEMA 4: FALTA CACHE (REDIS)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Severidad: MEDIA
Impacto: Consultas repetidas a BD
Oportunidad: Datos semi-estáticos (tarifas, zonas)

Datos Que Pueden Cachearse:
  - Tarifas (Zona A → Zona B): Cambia muy pocas veces
    Validez: 1 día
    Beneficio: -90% consultas a BD para este tipo
  
  - Zonas Geográficas: Prácticamente estáticas
    Validez: 7 días
    Beneficio: -95% para listados de zonas
  
  - Datos de Usuario: Vida de sesión
    Validez: 24 horas o logout
    Beneficio: -50% queries a usuarios

Impacto Estimado Sin Caché:
  1000 solicitudes/día de cotización
  × 2 queries por solicitud (zona_origen + zona_destino)
  = 2000 queries+de BD

Impacto Con Caché:
  1000 solicitudes/día
  × 0 queries (hit de caché)
  = 0 queries (después del warmup)

Solución (6-8 horas):

  1. Instalar Redis:
     # Windows: Descargar Redis MSI
     # Linux: sudo apt-get install redis-server
     # Docker: docker run -d -p 6379:6379 redis:latest
  
  2. Instalar cliente:
     pip install redis
  
  3. Implementar cache layer (infrastructure/cache/redis_cache.py):
     
     from redis import Redis
     import json
     
     class RedisCache:
        def __init__(self, host='localhost', port=6379):
          self.client = Redis(host=host, port=port, decode_responses=True)
        
        def get(self, key: str):
          data = self.client.get(key)
          return json.loads(data) if data else None
        
        def set(self, key: str, value, ttl: int = 86400):
          self.client.setex(key, ttl, json.dumps(value))
        
        def delete(self, key: str):
          self.client.delete(key)
  
  4. Usar en repositorio de tarifas:
     
     from infrastructure.cache.redis_cache import RedisCache
     
     class TarifaRepository(BaseRepository[Tarifa]):
        def __init__(self, session, cache: RedisCache):
            super().__init__(session)
            self.cache = cache
        
        def buscar_tarifa(self, origen: str, destino: str) -> Optional[Tarifa]:
            cache_key = f"tarifa:{origen}:{destino}"
            
            # Intenta caché
            cached = self.cache.get(cache_key)
            if cached:
                return Tarifa(**cached)
            
            # Query a BD
            tarifa = super().buscar_tarifa(origen, destino)
            
            # Guarda en caché (1 día)
            if tarifa:
                self.cache.set(cache_key, tarifa.dict(), ttl=86400)
            
            return tarifa

Beneficio:
  ✓ Reducción de carga en BD: -80-90%
  ✓ Mejor rendimiento de API: -50-70% latencia
  ✓ Escalabilidad mejorada
  ✓ Mejor experiencia de usuario

      # Busca patrones tipo "de/desde ciudad a ciudad"
      patron_de_a = r'(?:de|desde|en|salida)\s+(\w+)\s+(?:a|hacia|destino|llegar)\s+(\w+)'
      match = re.search(patron_de_a, texto_lower)

PROBLEMA 7: RATE LIMITING AUSENTE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Severidad: MEDIA
Impacto: Vulnerable a abuse/DOS
Endpoints Afectados: /chatbot/cotizar (público), /auth/login

Riesgo:
  Someone could do:
  for i in range(100000):
    requests.post('http://localhost:8000/api/v1/chatbot/cotizar')
  → Crash del servidor
  → Consumo alto de BD
  → Denegación de servicio

Solución (3 horas):

  1. Instalar SlowAPI:
     pip install slowapi
  
  2. Configurar en app/main.py:
     
     from slowapi import Limiter
     from slowapi.util import get_remote_address
     from slowapi.errors import RateLimitExceeded
     
     limiter = Limiter(key_func=get_remote_address)
     app.state.limiter = limiter
     
     @app.exception_handler(RateLimitExceeded)
     async def rate_limit_handler(request, exc):
        return JSONResponse(
            status_code=429,
            content={
                "error": "Too many requests",
                "detail": "Has superado en número de solicitudes. Intente más tarde.",
                "retry_after": 60
            }
        )
  
  3. Aplicar a endpoints:
     
     @router.post("/cotizar")
     @limiter.limit("10/minute")  # 10 cotizaciones por minuto por IP
     async def procesar_cotizacion(request: Request, data: CotizacionRequest):
        ...
     
     @router.post("/login")
     @limiter.limit("5/minute")  # 5 intentos por minuto
     async def login(credentials: LoginRequest):
        ...

rate_limits Recomendados:
  - /chatbot/cotizar: 10 requests/minute por IP
  - /auth/login: 5 requests/minute por IP
  - /auth/register: 3 requests/minute por IP
  - Endpoints internos: 100 requests/minute por token
